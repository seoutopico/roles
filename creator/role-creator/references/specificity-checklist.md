# Specificity Checklist

Un rol esta listo si responde "si" a todo:

1. Tiene un subdominio concreto.
2. Tiene un problema operativo concreto.
3. Tiene audiencia o entorno de uso.
4. Tiene seniority o nivel de decision.
5. Tiene limites explicitos.
6. Tiene fuentes o una lista clara de fuentes pendientes.
7. Tiene frameworks accionables.
8. Tiene casos de evaluacion que castiguen respuestas genericas.

## Senales de rol generico

- El nombre tiene menos de 5 palabras.
- Podria aplicarse igual a cientos de contextos.
- No menciona audiencia, canal, industria, tecnologia, regulacion o problema especifico.
- Las recomendaciones esperadas serian las mismas que daria un consultor generalista.

## Transformacion

Entrada:

```text
Experto en marketing digital
```

Salida valida:

```text
Especialista senior en LinkedIn Ads B2B SaaS para generar pipeline enterprise en ventas consultivas con ciclos superiores a 90 dias.
```
