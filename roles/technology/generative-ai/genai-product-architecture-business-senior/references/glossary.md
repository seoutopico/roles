# Glosario

Terminos clave del rol.

- Agente: sistema que usa un modelo para planificar o decidir pasos, llamar herramientas, mantener estado y avanzar hacia un objetivo con algun grado de autonomia.
- Arquitectura de contexto: forma en que el sistema selecciona, prepara, limita y entrega informacion relevante al modelo.
- Ahorro capturable: beneficio economico que se puede convertir en menor coste, mayor capacidad util o ingresos; no equivale a minutos teoricos ahorrados.
- Confabulacion: salida falsa o erronea presentada con aparente seguridad por un sistema generativo.
- Evals: pruebas sistematicas que miden calidad, seguridad, coste, latencia o impacto de un sistema de IA.
- Fallback: comportamiento alternativo cuando el modelo no debe responder, no alcanza confianza suficiente o falla un componente.
- Fine-tuning: ajuste de un modelo con ejemplos para modificar comportamiento, formato o rendimiento en tareas especificas.
- Guardrail: control preventivo, detectivo o correctivo que limita entradas, salidas o acciones del sistema.
- Human-in-the-loop: intervencion humana obligatoria antes, durante o despues de una salida o accion del sistema.
- Model routing: seleccion dinamica de modelo segun tarea, coste, latencia, riesgo o calidad requerida.
- Observabilidad: trazas, logs, metricas y revisiones que permiten entender rendimiento, costes, fallos y riesgos en produccion.
- Overreliance: confianza excesiva en salidas de IA sin revision critica adecuada.
- Prompt injection: intento directo o indirecto de manipular el comportamiento del modelo mediante instrucciones maliciosas o no autorizadas.
- RAG: retrieval-augmented generation; patron que recupera contexto externo autorizado y lo entrega al modelo para responder.
- Red teaming: pruebas adversariales para detectar fallos de seguridad, seguridad del usuario, privacidad, sesgos o usos indebidos.
- ROI: retorno sobre inversion; en IA generativa debe calcularse con costes recurrentes, supervision, soporte, cambio y riesgo.
- Tool calling: capacidad del modelo o sistema para invocar funciones, APIs o herramientas externas con parametros estructurados.
- Total cost of ownership: coste total de construir, operar, evaluar, mantener y gobernar una solucion, no solo coste de tokens.
