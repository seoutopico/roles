# Frameworks

Frameworks accionables para analizar proyectos de IA generativa desde datos, negocio y narrativa ejecutiva.

## 1. Cadena Por Que -> Para Que -> Valor

### Uso

Cuando el usuario tiene una idea, demo, piloto o tecnologia y necesita justificarla ante negocio.

### Inputs

- Proyecto o caso de uso.
- Usuario o area afectada.
- Dolor actual.
- Objetivo declarado.
- Datos disponibles.

### Pasos

1. Formular el "por que": problema o oportunidad concreta que existe aunque no hubiera IA.
2. Formular el "para que": decision, tarea o workflow que se quiere mejorar.
3. Formular el "valor": resultado observable en negocio, operacion, riesgo, experiencia o aprendizaje.
4. Identificar el baseline: que ocurre hoy, cuanto cuesta, cuanto tarda, que error genera o que oportunidad se pierde.
5. Declarar la hipotesis GenAI: por que generativo y no BI, reglas, automatizacion clasica o proceso manual.
6. Definir criterio de decision: que dato haria seguir, pausar o replantear.

### Output

- Una frase de negocio defendible: "Hacemos X para cambiar Y, medido por Z, porque hoy ocurre W".

## 2. Arbol De Objetivos GenAI

### Uso

Cuando hay que convertir una iniciativa ambigua en objetivos medibles.

### Inputs

- Objetivo estrategico.
- Proceso afectado.
- Usuarios.
- Datos historicos.
- Restricciones de riesgo y compliance.

### Pasos

1. Outcome estrategico: crecimiento, margen, retencion, eficiencia, riesgo, calidad, satisfaccion, aprendizaje o diferenciacion.
2. Resultado de negocio: KPI que importa al comite.
3. Resultado operativo: tiempo, coste, errores, throughput, calidad, conversion, tickets, ciclo o backlog.
4. Cambio de comportamiento: que hara distinto el usuario o el equipo.
5. Metrica de adopcion: uso real, frecuencia, cobertura, recurrencia, abandono y excepciones.
6. Metrica de calidad GenAI: precision, completitud, factualidad, utilidad, seguridad, consistencia, latencia y coste por tarea.
7. Criterio de escala: umbral minimo, riesgo aceptable, owner, proceso y presupuesto.

### Output

- Arbol de metricas conectadas y no decorativas.

## 3. Scorecard De Proyecto GenAI

### Uso

Para decidir si un proyecto debe seguir, pausarse o replantearse.

### Dimensiones

- Valor potencial: tamano del problema, frecuencia, margen, riesgo evitado o capacidad liberada.
- Evidencia: baseline, datos reales, usuarios observados, resultados de prueba y comparacion contra alternativa.
- Encaje GenAI: necesidad de lenguaje, razonamiento, generacion, resumen, clasificacion flexible, busqueda semantica o agentes.
- Viabilidad de datos: acceso, calidad, permisos, estructura, freshness, trazabilidad y sensibilidad.
- Adopcion: friccion del workflow, incentivos, confianza, formacion, owner y soporte.
- Operacion: coste variable, latencia, monitoreo, evals, rollback, seguridad y mantenimiento.
- Riesgo: inexactitud, privacidad, PI, sesgo, compliance, reputacion y dependencia de proveedor.
- Narrativa: capacidad de explicar la decision en una frase honesta y medible.

### Output

- Decision: escalar, pilotar, medir, reducir scope, redisenar o abandonar.

## 4. Matriz Evidencia / Supuesto / Apuesta

### Uso

Cuando una presentacion mezcla datos, intuiciones y promesas.

### Pasos

1. Marcar como hecho lo que esta observado con datos internos o fuente verificable.
2. Marcar como inferencia lo que se deduce razonablemente de hechos.
3. Marcar como supuesto lo que debe ser cierto para que el caso funcione.
4. Marcar como apuesta lo que puede ser estrategicamente deseable aunque aun no este probado.
5. Convertir cada supuesto critico en medicion o experimento.

### Output

- Tabla con columna "grado de certeza" y "decision que bloquea".

## 5. Narrativa Ejecutiva SO-WHAT

### Uso

Para convertir analisis en slides de comite.

### Pasos

1. Situacion: que problema de negocio existe ahora.
2. Observacion: que muestran datos, usuarios, proceso o piloto.
3. Implicacion: por que importa para margen, crecimiento, riesgo, velocidad o experiencia.
4. Decision requerida: que debe aprobar, pausar o pedir el comite.
5. Condicion de exito: como se sabra si funciono.

### Output

- Titular de slide: conclusion + implicacion + decision.

## 6. Filtro Anti-Humo

### Uso

Antes de entregar frases para PowerPoint.

### Preguntas

- La frase dice que cambia en el negocio o solo suena moderna?
- Tiene sujeto, verbo, metrica y condicion?
- Diferencia piloto de impacto?
- Declara el principal supuesto?
- Evita "transformar", "potenciar" y "optimizar" sin objeto concreto?
- Se puede defender ante Finanzas, Tecnologia, Legal y Operaciones?

### Output

- Frase ejecutiva sobria, medible y defendible.
