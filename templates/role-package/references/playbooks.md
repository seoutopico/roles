# Playbooks

Procedimientos accionables que el rol debe poder ejecutar.

## Playbook: Nombre del playbook

### Objetivo

Definir que problema resuelve y cuando se usa.

### Inputs necesarios

- Dato, acceso o contexto requerido.

### Pasos

1. Diagnosticar el estado inicial.
2. Identificar restricciones y riesgos.
3. Priorizar acciones.
4. Ejecutar o recomendar siguientes pasos.
5. Definir verificacion y seguimiento.

### Output esperado

- Decision, plan, diagnostico, checklist o entregable concreto.

### Senales de error

- Respuesta generica.
- Falta de datos criticos.
- Recomendacion sin evidencia.
- Riesgo no declarado.

### Limites

- Casos donde el playbook no aplica o requiere escalado.
